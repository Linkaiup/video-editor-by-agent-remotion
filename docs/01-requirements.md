# Remotion Harness Agent - 需求分析文档

## 1. 项目背景

基于 Remotion 框架构建一个 AI Agent，通过自然语言对话实现视频编辑功能。

## 2. 核心需求

### 2.1 功能需求

#### 2.1.1 基础功能（P0）
- [ ] **自然语言输入**：用户通过对话方式描述需求
- [ ] **意图识别**：准确理解用户想做什么（创建、编辑、查询等）
- [ ] **视频创建**：根据素材生成 Remotion 组件并渲染为 MP4
- [ ] **素材处理**：支持本地图片、视频等素材
- [ ] **基础特效**：淡入淡出、缩放、旋转等
- [ ] **视频预览**：在浏览器中预览 Remotion 组件
- [ ] **视频导出**：渲染为 MP4 文件

#### 2.1.2 进阶功能（P1）
- [ ] **多轮对话**：支持连续对话和上下文理解
- [ ] **视频编辑**：修改已有视频的参数、特效
- [ ] **转场效果**：多个片段之间的转场
- [ ] **文字动画**：添加文字和动画效果
- [ ] **音频支持**：添加背景音乐和音效
- [ ] **模板系统**：预设模板快速生成

#### 2.1.3 高级功能（P2）
- [ ] **批量处理**：一次处理多个素材
- [ ] **智能推荐**：根据素材推荐特效和转场
- [ ] **版本管理**：保存和管理多个版本
- [ ] **协作功能**：多人协作编辑
- [ ] **插件系统**：扩展自定义特效和转场

### 2.2 非功能需求

#### 2.2.1 性能需求
- 意图识别响应时间 < 3s
- 视频渲染速度 >= 实时（1080p@30fps）
- 支持 4K 分辨率输出
- 并发处理能力 >= 5 个任务

#### 2.2.2 可用性需求
- 对话理解准确率 >= 90%
- 中英文双语支持
- 友好的错误提示
- 完整的使用文档

#### 2.2.3 可维护性需求
- 模块化架构设计
- 完整的中文注释
- TypeScript 类型安全
- 单元测试覆盖率 >= 80%

#### 2.2.4 可扩展性需求
- 插件化特效系统
- 可配置的 LLM 后端
- 支持自定义 Remotion 组件
- RESTful API 接口

## 3. 用户场景

### 3.1 场景 1：快速创建视频
**用户**：内容创作者  
**目标**：快速将图片制作成视频  
**流程**：
1. 用户：创建一个 10 秒的视频，使用这张图片
2. Agent：识别意图 → 处理图片 → 生成代码 → 渲染视频
3. 用户：视频在哪？
4. Agent：显示视频路径和预览

### 3.2 场景 2：添加特效
**用户**：视频编辑师  
**目标**：为视频添加专业特效  
**流程**：
1. 用户：给刚才的视频添加缩放效果
2. Agent：读取上下文 → 修改组件 → 重新渲染
3. 用户：不对，我想要从小到大的缩放
4. Agent：调整参数 → 重新渲染

### 3.3 场景 3：批量处理
**用户**：自媒体运营  
**目标**：批量制作社交媒体视频  
**流程**：
1. 用户：用这 10 张图片分别创建 10 个视频，每个 5 秒，添加淡入淡出
2. Agent：并发处理 → 生成 10 个视频
3. 用户：第 3 个和第 7 个需要重新做
4. Agent：重新处理指定的视频

## 4. 技术约束

### 4.1 必须使用的技术
- Remotion 4.x（视频渲染引擎）
- React + TypeScript（组件开发）
- Node.js 18+（运行环境）
- LLM API（意图识别和规划）

### 4.2 技术方案确定 ✅
- **LLM**：DeepSeek（继续使用）
- **素材存储**：本地文件系统
- **会话存储**：文件系统（JSON）
- **部署方式**：CLI + Web UI
- **前端框架**：React（复用 Remotion）
- **后端**：Node.js + Express

## 5. 现有问题分析

### 5.1 架构问题
- ❌ 缺少多轮对话的上下文管理
- ❌ 没有持久化存储（会话、模板、历史）
- ❌ 意图识别过于简单，容易识别错误
- ❌ 缺少错误恢复和重试机制

### 5.2 功能缺失
- ❌ 无法编辑已有视频
- ❌ 不支持多个素材组合
- ❌ 缺少模板系统
- ❌ 没有音频处理能力
- ❌ 转场效果不完整

### 5.3 用户体验问题
- ❌ 意图理解错误时提示不友好（"您是想 unknown 吗"）
- ❌ 视频生成失败后无法重试
- ❌ 没有进度反馈（长视频渲染时）
- ❌ 缺少视频预览功能

### 5.4 代码质量问题
- ❌ 类型安全性不足（大量 any 和可选链）
- ❌ 错误处理不完善
- ❌ 缺少单元测试
- ❌ 代码生成过于依赖 LLM（不稳定）

## 6. 需求优先级

### Phase 1 - MVP（当前阶段）✅
- ✅ 基础对话能力

- ✅ MP4 导出

  - 多轮完善和调整机制
  - 自动校验生成结果
  - 错误自动重试
- 🎯 完善多轮对话和上下文管理
- 🎯 视频编辑能力（修改已有视频）
- 🎯 多素材组合
- 🎯 更丰富的特效和转场
- 🎯 模板系统
- 🎯 **Web UI 界面**
  - Agent 对话界面
  - 素材上传功能
  - 素材列表展示
  - 视频预览
- 🎯 完整的测试覆盖

### Phase 3 - 专业版
- 音频支持
- 批量处理
- Web UI
- API 服务
- 插件系统
